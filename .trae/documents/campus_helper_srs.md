# 软件需求规格说明书 (SRS) - 校园互助平台

## 1. 引言

### 1.1 目的

本文档详细描述了“校园互助平台”的功能需求、性能需求、外部接口需求以及质量属性。本文档旨在作为开发、测试及验收的基准。

### 1.2 范围

本项目包含：

* 基于 Taro 的多端小程序（微信、支付宝、H5）。

* 基于 React 的管理后台系统。

* 基于 Supabase 的后端服务与数据库。

***

## 2. 系统功能需求

### 2.1 身份验证与权限 (Auth)

* **F-01**: 支持学号/邮箱注册及登录。

* **F-02**: 支持微信一键登录（小程序端）。

* **F-03**: 基于角色 (Role-based) 的访问控制：`student`（普通学生）, `admin`（系统管理员）。

### 2.2 任务管理 (Task Management)

* **F-04**: 任务发布：标题、描述、类型、酬金、位置（地图选点）、截止时间。

* **F-05**: 任务审核：管理员后台对新发布的任务进行内容审核。

* **F-06**: 任务列表与搜索：支持按距离、酬金、类型、时间排序和筛选。

* **F-07**: 任务状态流转：`pending` -> `published` -> `accepted` -> `completed` / `cancelled`。

### 2.3 信用分体系 (Credit System)

* **F-08**: 初始信用分 100 分。

* **F-09**: 自动加减分：成功完成任务加分，违约/被举报核实扣分。

* **F-10**: 权限约束：信用分低于 60 分禁止接单。

### 2.4 地图与定位 (LBS)

* **F-11**: 首页展示附近任务标记。

* **F-12**: 支持小程序原生定位功能。

* **F-13**: 点击标记查看任务简略信息并支持导航。

### 2.5 支付与结算 (Payment)

* **F-14**: 酬金托管：发布者发布时预支付酬金。

* **F-15**: 自动结算：任务完成后，酬金自动转入接单者账户（扣除平台手续费）。

***

## 3. 非功能需求

### 3.1 性能需求

* **P-01**: 小程序首页加载时间不超过 2 秒。

* **P-02**: 地图标记渲染支持 100+ 同时展示而不卡顿。

* **P-03**: API 接口响应时间不超过 500ms。

### 3.2 可靠性与安全性

* **S-01**: 数据加密：所有用户信息加密存储。

* **S-02**: RLS 策略：严格限制用户只能访问和修改属于自己的数据。

* **S-03**: 并发处理：防止同一任务被多人接单。

### 3.3 用户界面规范

* **U-01**: 小程序端遵循微信小程序设计指南。

* **U-02**: 管理端支持主流浏览器（Chrome, Edge, Safari）。

***

## 4. 外部接口需求

* **接口 1**: 高德地图小程序 SDK 接口。

* **接口 2**: 微信支付 / 支付宝支付接口。

* **接口 3**: Supabase Auth & Database SDK。
